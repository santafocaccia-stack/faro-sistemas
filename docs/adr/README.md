# Architecture Decision Records

> Acá viven las decisiones arquitectónicas importantes.
> Cada decisión grande se documenta con un ADR para que el "por qué" no se pierda.

## Cómo escribir un ADR

1. Crear archivo `NNN-titulo-corto.md` (NNN = número correlativo)
2. Usar el [template](./000-template.md)
3. Estado posible: `Propuesto` → `Aceptado` / `Rechazado` → `Reemplazado por NNN`
4. Una vez aceptado, no se edita el contenido — se crea uno nuevo que lo reemplaza
